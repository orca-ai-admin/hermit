# Corrections Log

Track every correction from the human. Each entry becomes data for the self-improvement engine.

## Template

### YYYY-MM-DD

- [HH:MM] Description of what went wrong
  - **Type**: communication | accuracy | behavior | infrastructure
  - **Severity**: minor | moderate | major
  - **Pattern**: (recurring pattern name, if any)
  - **Root cause**: Why it happened
  - **Fix applied**: What was done immediately
  - **Structural fix needed**: Yes/No — if yes, becomes experiment candidate

## Categories

| Type | Examples |
|------|---------|
| communication | Message bursts, wrong tone, over-explaining |
| accuracy | Wrong information, misread data, false claims |
| behavior | Permission-seeking, tangents, claiming inability |
| infrastructure | Service outage, broken config, failed deployment |

## Tips

- Record the human's exact words when possible
- Note the pattern name if it matches a known pattern
- Record successes too — confirms what's working
- Use absolute dates, not relative ("2026-03-25" not "yesterday")
